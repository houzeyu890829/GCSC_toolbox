# GCSC_toolbox
Matlab code for efficient source identification and parameter estimation of DNAPL-contaminated aquifer using Sensitivity-based Dynamic PSO with Metropolis Criterion and Genetic evolution-kernel extreme learning machine  
Firstly, the folders of ‘gaot’ and ‘gatbx1.3’ should been added into the Matlab paths to make the Genetic evolution-kernel extreme learning machine run. The main program for ‘Sensitivity-based Dynamic PSO with Metropolis Criterion’ inversion was named 'P_S_PSO.m' in the folder of 'SD-PSO with Metropolis Criterion', and 'P_S_PSO_quicktest.m' was created to conduct the quick-test. The established forward GE-WKELM surrogate model had 12 input variables and 8 output variables, which was set as a function ‘KELMdnapl’ in folder of 'SD-PSO with Metropolis Criterion', and 200 training samples and 20 testing samples were listed in 'Syangben.txt'. The inverse GE-WKELM surrogate model had 8 input variables and 12 output variables, the programs for training and testing were in the folder ‘Inverse-GE-WKELM’.
